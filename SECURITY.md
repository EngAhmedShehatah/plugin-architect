# Security Policy

## Supported Versions

Only the latest released version of plugin-architect is actively supported with security fixes.

| Version | Supported |
|---------|-----------|
| latest  | ✅        |
| older   | ❌        |

## Reporting a Vulnerability

If you discover a security vulnerability in plugin-architect, **please do not open a public GitHub issue.**

Instead, report it privately via one of these channels:

- **GitHub Private Advisory**: [Report a vulnerability](https://github.com/EngAhmedShehatah/plugin-architect/security/advisories/new)
- **Email**: [Author Email](engahmedshehatah@gmail.com)

### What to include

- A clear description of the vulnerability
- Steps to reproduce it
- The potential impact
- Any suggested fix (optional)

### What to expect

- **Acknowledgement** within 48 hours
- **Status update** within 7 days (confirmed, declined, or needs more info)
- **Fix timeline** communicated once the issue is confirmed
- **Credit** in the release notes if you wish to be acknowledged

## Scope

This policy covers the plugin-architect repository and its core plugin. It does not cover third-party tools, plugins, or MCP servers that plugin-architect may reference or integrate with.
