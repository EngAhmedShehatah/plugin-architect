# Contributing to plugin-architect

Thank you for your interest in plugin-architect!

## Current Contribution Model

This project is currently in early-stage development and operates under a **maintainer-only** model. The core architecture is still being defined, and we are not accepting unsolicited pull requests at this time.

This will change once the project reaches a stable release. We will update this file and announce it when external contributions are welcome.

## How You Can Help Right Now

- **Open an issue** to report a bug or suggest a feature
- **Start a discussion** if you have a question or idea
- **Star the repo** to show support and help others discover the project

## For Invited Collaborators

If you have been invited as a collaborator, please follow these conventions:

### Branch Naming

- `feat/<short-description>` — new features
- `fix/<short-description>` — bug fixes
- `chore/<short-description>` — maintenance, config, tooling
- `docs/<short-description>` — documentation only

### Commit Style

Follow [Conventional Commits](https://www.conventionalcommits.org/):

- `feat: add tech-stack-detector agent`
- `fix: resolve schema-scanner monorepo detection`
- `chore: bump version to 1.1.0`
- `docs: update CONTRIBUTING guide`

### Pull Request Rules

- PRs must target `main`
- At least 1 approving review required before merge
- Keep PRs focused — one concern per PR
- Link to the relevant issue if applicable

## Code of Conduct

Please read our [Code of Conduct](CODE_OF_CONDUCT.md) before engaging with this project.
